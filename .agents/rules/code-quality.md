# Code Generation and Reasoning Rules
- **Activation**: Always On

## Constraints
* **Multi-Step Reasoning**: Think step-by-step. Draft your logic and evaluate edge cases inside a `<thinking>` block before writing a single line of code.
* **No Placeholders**: Do not use code placeholders or truncate the code (e.g., avoid `// TODO: implement later`). Provide the full, functional implementation.
